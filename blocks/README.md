# blocks/ · 装配块（Block）

> 江河隐喻：河床、河堤。理论身份：纯机制（mechanism）。
> 纪律来源：`.kiro/steering/afp-core.md`。

## 是什么

装配块是**与业务无关的最小组件**，像 lodash 一样可全球复用、几乎不变（仅 bug/升级才动）。

## 硬规矩

- **必须是纯机制**：不含任何业务策略。"用户注册"不是装配块，它是装配流。
- **强契约**：每个块必须带 I/O Schema + 语义化版本 + 测试用例。
- **算法住在这里**：配置里不准写的算法/复杂判定/动态循环，全部封进装配块当黑盒，由单测覆盖。
- **依赖白名单**：块的外部依赖必须在白名单内、可审计。

## 来源约定

转接件里反复出现的"字段映射/类型转换"应被挖出来、升级成装配块（见 skill `afp-extract-block`）——它们本就是纯机制。
