# adapters/ · 转接件（Adapter）

> 江河隐喻：都江堰（按需分流）。理论身份：业务适配 / 防腐层。
> 纪律来源：`.kiro/steering/afp-core.md`。

## 是什么

转接件是装下"业务善变"的地方。它只做**按需分流、字段映射、类型转换**——不改河床（装配块）、不改河水（数据）。各业务自己写，天然不全球复用。

## 硬规矩

- **只组合/模板化，禁止继承**。继承=脆弱基类，会把刚甩掉的耦合请回来。
- **越缩越小**：反复出现的纯机制要持续挖出去升级成装配块，转接件最后只剩不可再约简的业务决定。
- **套用重复检测回路**：和装配块同一套，防止"代码复制"变成"配置复制"。
