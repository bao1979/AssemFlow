# configs/ · 配置（Config）与装配流（Flow）

> 江河隐喻：河图（接线蓝图）/ 流动的河。理论身份：策略声明 / 装配接线。
> 纪律来源：`.kiro/steering/afp-core.md`。

## 是什么

配置是**接线蓝图**：声明哪些装配块经哪些转接件、按什么结构拼成一条**装配流**。设计期由 AI 产、人审。装配流就是一份（或一组）配置定义出来的完整业务流。

## 硬规矩（红线）

- **算法不入配置**。配置只准表达"结构、接线、声明式策略"。在配置里写算法 = 造一门很烂的编程语言。
- **静态可枚举**：
  - 允许：对已声明的有限域谓词（布尔/枚举）分支、对带类型集合做有界 `for_each`。
  - 禁止：计算出来的/依赖运行时数据的谓词、深层嵌套条件、动态边界循环——这些封进装配块。
  - 判据："VIP 走 A 否则走 B"留配置；复杂判定入块。
- **配置即图**：配置结构必须能被引擎静态分析（where-used、死配置、影响面、类型校验）。

## 切细优先

拿不准粒度时切细。小块可用具名配置打包成"复合装配流"（组合可逆，分解破坏性）。

## 配置格式（已定案）

- 配置源用 **JSON**（开发期 JSONC 允许注释）+ **JSON Schema** 校验。
- 纯数据格式，物理上塞不进可执行逻辑，从形态上守住"算法不入配置"。
- 不用 HTML/JS/代码做配置源；配置的可视化由引擎渲染成 HTML/SVG 派生视图，供人审。
