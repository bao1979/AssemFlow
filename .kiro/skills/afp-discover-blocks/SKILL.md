# 发现装配块（Discover Blocks）

破解软件复用 30 年头号死结——"找不到合适的组件"。LLM 的语义匹配正是解法。

## 何时用

- 用户用自然语言描述一段业务流程或需求
- 准备产新配置 / 新装配流之前
- 用户问"有没有现成的块/转接模式能复用"

## 步骤

1. **拆需求为机制原语**：把业务描述拆成一串"与业务无关的纯机制"动作（如"发邮件""写审计日志""校验邮箱格式""幂等去重"）。注意区分：纯机制 → 找装配块；业务策略 → 留给转接件/配置。
2. **检索 `blocks/`**：按机制语义匹配已有装配块，读其 `schema` 确认 I/O 契约是否吻合。
3. **检索转接模式 / 配置模板**：在 `configs/`、`adapters/` 找可复用的起手式。
4. **输出三类清单**：
   - 可直接复用的装配块（含版本与契约）
   - 需要新写的装配块（纯机制缺口）
   - 落在"业务策略"侧、应进配置/转接件的部分
5. **缺口处理**：缺的纯机制 → 提议新建装配块（走强契约）；属业务策略 → 不要造块，留给配置/转接件。

## 红线

- 不要把业务策略误判为装配块（"用户注册"是装配流，不是块）。
- 不臆造不存在的块/包名——只报 `blocks/` 里真实存在的；防 slopsquatting。
- 发现结果供设计期产配置用，不在运行期做任何 AI 匹配。
