# 挖块收敛（Extract Block）

把易变性赶进转接件后，转接件自己会繁殖、重复——熵从河床搬到了闸口，不自动收敛。本技能是收敛回路。

## 何时用

- 转接件越写越大、越写越多
- 多个转接件里出现重复的字段映射/类型转换
- 定期治理（把"配置复制"成本压回去）

## 步骤

1. **找重复纯机制**：扫描 `adapters/`，定位反复出现的"字段映射/类型转换"——它们本就是纯机制，错放在了转接件里。
2. **判定纯度**：确认这段逻辑与业务策略无关（不依赖具体业务规则）。掺了策略的不要挖，留在转接件。
3. **升级为装配块**：抽到 `blocks/`，配 I/O Schema + 语义化版本 + 测试，依赖走白名单。
4. **回填转接件**：让原转接件改为引用新块，删掉重复代码。转接件应明显变小。
5. **更新配置/契约**：若块的引入改变了接线，同步配置；走影响面分析确认波及范围。
6. **版本纪律**：v1.0 后若动到已被依赖的块，走语义化版本 + 别名/多版本共存，必要时 codemod 迁移配置。

## 红线

- 只挖纯机制；业务策略不许进装配块。
- 转接件之间用组合，不用继承。
- 目标是"转接件越缩越小，最后只剩不可再约简的业务决定"。
- 别把"代码复制"换成"配置复制"——挖块是真正的收敛动作。
