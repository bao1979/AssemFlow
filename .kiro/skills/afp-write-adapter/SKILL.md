# 写转接件（Write Adapter）

转接件是都江堰：把易变性从装配块（河床）里挤出去，锁进一个局部、廉价、可丢弃的地方。

> 路径说明：下文 `blocks/` `adapters/` 指**构件仓这一角色**，按当前载体解析——验证期在 `experiments/expNN-*/src/` 下，生产期在仓库根。详见 `afp-core.md`「构件路径约定」。

## 何时用

- 装配块的 I/O Schema 与业务需求对不上，需要映射/转换
- 需要"按需分流"（同一入口按业务规则走不同块）
- 配置接线时发现相邻块契约不齐

## 步骤

1. **先问能否不写**：缺口是不是其实该入配置（枚举路由）或该入装配块（纯机制）？转接件只装"不可再约简的业务决定 + 适配胶水"。
2. **只做三件事**：字段映射、类型转换、按需分流。不在转接件里写算法（算法进块）。
3. **组合，不继承**：用组合/模板化复用，**禁止让转接件继承转接件**（脆弱基类会把耦合请回来）。
4. **保持最小**：写完回看——里面有没有"反复出现的字段映射/类型转换"？有 → 用 `afp-extract-block` 挖出去升级成装配块。
5. **配测试**：转接件也要测，套用与装配块同一套重复检测回路。

## 红线

- 禁止继承。
- 不改装配块、不改数据，只做适配分流。
- 转接件是局部资产，不奢望全球复用。
- 越缩越小是健康信号；越长越大是要挖块的警报。
