# 文档同步规范（单一真相源模型）

> 设计目标：人精力有限、不可能手动同步；AI 同步必须**省 token、不易出错、文档恒新**。
> 实现手段：**结构上消灭副本**——每条信息只存一处，AI 只改一处，天然无漂移。**不靠任何代码/脚本同步**（当前无引擎）。

## 铁律：一条信息只存一处

禁止把同一信息抄到多个文件。需要在别处引用时，**写一句指向真相源的链接**，不复制内容。

## 四个文件的职责与变化速率

| 文件 | 层级 | 存什么 | AI 何时更新 |
| :--- | :--- | :--- | :--- |
| `docs/ai/state.json` | 高频 | 进度、模块状态、活跃决策、下一步、开放问题、已知问题 | **几乎每次对话**，只改变动字段 |
| `docs/ai/system-design.md` | 中频 | 架构、五元构件模型、目录结构、技术栈论证 | 仅当架构/技术栈/目录变化 |
| `docs/ai/decisions-archive.json` | 低频 | 已退场（无后续约束力）的决策 | 决策退场时从 state 移入 |
| `README.md`（仓库根） | 极低频 | 对外门面：定义、五元构件、架构图、导航 | 仅范式定义级变化 |

## AI 的同步动作（极简）

完成一项工作后：

1. **更新 `state.json`**：改 `updatedAt`（当天日期）+ 受影响的字段（`note` / `modules` / `nextSteps` / `decisions` / `openQuestions` / `knownIssues`）。**只改动变化的字段，不重写全文**——省 token。
2. **仅当架构/技术栈/目录真的变了**，才同时更新 `system-design.md`。
3. **决策退场时**（某 final 决策不再有后续约束力），把它从 `state.json.decisions` 移到 `decisions-archive.json`。
4. **基本不动** `README.md`（根）——它只在范式定义级变化时更新。

> 绝大多数对话只触动 `state.json` 一个文件。这就是"省 token、不易错"的来源。

## state.json 瘦身规则

- `decisions` 只保留**对当前/后续开发有约束力**的 final 决策；纯历史动作归档到 `decisions-archive.json`。
- `knownIssues` 只保留未解决项。
- `modules` 里 Done 模块用一句话摘要；InProgress/Planned 保留必要结构。
- 不维护"改动文件清单"这类高熵、低价值字段（git 已记录）。

## 不要做的事

- 不要新建与上述四文件职责重叠的"仪表盘""路线图""模块地图"等副本文件——它们正是漂移的根源，已被本模型取消。
- 不要在 `system-design.md` 或 `README.md` 里抄 `state.json` 的进度/状态，只能链接。
- 不要等任务全部结束才更新；每完成一项就顺手更 `state.json`。
- 不要写脚本来"同步"文档——当前无引擎，且 SSOT 模型本就不需要同步。
