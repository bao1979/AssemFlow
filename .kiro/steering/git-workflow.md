# Git 操作规范

## 核心规则

- **默认只给提醒 + 可复制的指令，不自动执行任何 git 命令。**
- **仅当 Plucker518 明确点名**（如"帮我 git""你来提交""执行 git""push 一下"）时，才由 AI 执行 git。
- 不确定是否该执行时，按"不执行、只给指令"处理。

## 给指令时的要求

- 给出可直接复制运行的完整命令。
- 本机是 Windows PowerShell：**不支持 `&&` 连接符**，多条命令用分号 `;` 或分行写。
- 提交信息用**中文**，简洁、说明本次改动主旨。
- 涉及破坏性操作（force push、reset --hard 等）必须显著提示风险。

## 常用指令模板

```powershell
# 查看状态
git status

# 暂存全部改动
git add -A

# 提交
git commit -m "<简洁的改动说明>"

# 推送当前分支
git push
```

## 何时提醒提交

完成一组有意义的改动后（如一个模块、一组文档、一次实验产出），提醒 Plucker518"可以提交了"并附上指令，由他决定何时执行。
