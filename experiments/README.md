# experiments/ · 最小证伪实验

> 原则：证伪优先。任何一个实验失败，都比写一份漂亮 PRD 更有价值——它直接告诉你这条路在哪段是实的、在哪段是空的。
> 来源：`docs/装配流编程-可行性分析.md` · "建议的最小验证实验"。

## 三个实验

### exp01-sweet-spot · 甜区证伪

- **做法**：建全新极小项目（建议"注册 + 发邮件 + 写审计日志"服务），手写 3 个装配块 + 1 份配置，纯脚本拼装。
- **验证**：只改配置就能出"可编译 + 过冒烟"的代码。
- **为什么用全新项目**：零遗留、零叙事耦合，信号最纯。
- **判定**：成立 → 甜区假设为真。

### exp02-boundary · 10% / 边界证伪

- **做法**：故意挑 clawerworld 一个偏定制的真实需求，看 AFP 在哪一步崩、崩得多难看。
- **判定**：崩点位置决定 AFP 的适用边界。

### exp03-reuse · 复用证伪

- **做法**：让第二个业务流（如带短信验证的注册）复用第一个的装配块，**不改装配块代码、只加配置/转接件**。
- **判定**：能复用 → 复用假设为真；要改代码 → 粒度假设证伪。
