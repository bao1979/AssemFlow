# 范式验证路线图：Sokoban

> 这是一份**指导性路线图**，不是 spec。它给出 Sokoban 范式验证的全局共识、术语、参照物与 MVP 链总览。
> 每个 MVP 的可执行需求在各自独立 spec 里：`.kiro/specs/sokoban-mvp-{0..4}-*/`。
> 本文件是 `open-questions.md` 中 Q-028 的具体化，变化速率低（仅当全局共识变动时更新）。

## 为什么存在

本路线图是 AssemFlow 项目第二阶段的全局地图。

**目标升级**：经第一阶段三个最小证伪实验后，项目目标从"验证 AFP 是否成立"升级为：

> **寻找 AI agent 介入模式下的更优编程范式。**

AFP 是当前主推假设，但不是唯一约束。验证期允许混合其它范式（reducer / 状态机 / ECS / 行为树等），只要能让 **AI agent 与人协作开发** 更高效、更可靠、更可治理。每处非 AFP 范式必须显式标记（见"范式混合标记约定"）。

**载体**：Sokoban（推箱子）。已确定，不再讨论替代。理由：全民知名、规则一句话讲完、LLM 训练数据完整、回合制对应 AFP 甜区、每关=一份 ASCII 数据。

**推进方式**：拆成一条 **MVP 链**。每个 MVP 独立可交付、可单独得结论、可随时中止。引擎需要什么能力，由每个 MVP 的 Design 阶段按需暴露——**Requirements 只描述游戏行为与共识，不预先设计引擎内部结构**。这正契合"游戏已定、引擎想法未统一"的现状：边做边逼出引擎共识。

## Glossary

- **AFP**：装配流编程，本项目主推范式。详见 `.kiro/steering/afp-core.md`。
- **装配块 / 转接件 / 配置 / 数据 / 装配流**：AFP 五元构件，术语以 afp-core.md 为准。
- **MVP**：本路线图的最小可交付单元。MVP-0 ~ MVP-3 已完成推进；MVP-4 已归档（D-015）。
- **回合（Turn）**：玩家一次输入到游戏状态完成一次完整变换的最小执行单元。Sokoban 每次按键 = 一回合。
- **关卡（Level）**：一份描述初始网格状态的纯数据（ASCII）。
- **静态校验（check）**：不运行游戏时对配置 / 关卡数据的合法性分析。
- **K-STATE / K-LOOP**：长寿命状态演化 / 循环驱动两个议题，对应 Q-026 / Q-027。
- **非 AFP 范式**：验证期为跑通而引入的 reducer / 状态机 / 类 / 全局状态等，须用 `@paradigm` 标记。

## 参照物共识：Godot（借词汇，拒架构）

引擎该长什么样还没完全统一。为降低沟通成本，**借用 Godot 的概念词汇**作为讨论锚点——讨论"引擎要不要 X"时，用"Godot 的 X 是怎么做的，我们要 / 不要"来表达。

**借用的分层心智模型**：

| Godot 概念 | 本项目对应 | 说明 |
| :--- | :--- | :--- |
| Scene（场景） | 一个 Flow 配置 | 一关 ≈ 一个场景 |
| Resource（.tres 数据资源） | 关卡 ASCII / 配置 | **最贴的类比**：纯数据、可热换、不含逻辑 = AFP 的配置 |
| Input 映射（动作名） | 输入适配层 | 物理按键 → 动作名 → 一次回合输入，照搬这个思路 |
| `_process()` 主循环 | 引擎的回合驱动 | Godot 每帧 tick；我们每"按键"调一次 assemble（回合制） |

**明确拒绝的 Godot 特性**（它们是 AFP 的对立面）：

- **每帧 tick 的实时循环**：我们是回合驱动，不是帧驱动。
- **Node 继承树（面向对象）**：AFP 用组合 + 数据流，不用继承。
- **Signal（信号 / 事件总线）**：暂不引入，会把数据流搅乱。

> 一句话：Godot 是**词汇参照**，不是**架构参照**。

## 范式混合标记约定

验证期为跑通不限制引入非 AFP 范式，但**必须可机器扫描地标识**。凡是非 AFP 范式的代码文件，头部加统一标记块：

```ts
/**
 * @paradigm NON-AFP: reducer
 * @reason 撤销栈是时间维度状态，AFP 数据流表达不了历史
 * @afp-debt 未来可考虑把历史栈做成有状态块 / 收编进转接件
 */
```

- `@paradigm`：标明用了什么范式（reducer / state-machine / class / global-state 等）
- `@reason`：为什么 AFP 在这里不合适
- `@afp-debt`：未来可能的收编方向（呼应"非 AFP 的最终都尝试归入转接件"这一长期意图）

要求：`grep "@paradigm"` 能一键列出全仓所有非 AFP 处。验证期不强行归类为转接件；先如实记录它实际是什么，MVP 跑完再回头评估能否收编。

## 全链 Out of Scope（任何 MVP 都不做）

- **实时帧驱动游戏**：限定回合制。
- **引擎复用到其它游戏**：本阶段不验证，留待 v1.0 后。
- **网络对战 / 多人 / AI 对手 / 路径规划**：与范式验证无关。
- **3D 渲染 / 复杂动画 / 声音 / 美术资产**：渲染只做"够清楚就行"。
- **关卡可解性自动判定**：静态校验不含求解器（留扩展位，不实现）。
- **纯算法关卡自动生成**：让 LLM 产关卡属交付物 A；算法 PCG 不做。
- **真实生产部署 / 性能优化**：本路线图是研究产出，非商用产品。
