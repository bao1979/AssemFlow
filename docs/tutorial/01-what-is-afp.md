# 一张图看懂 AFP

> 本课目标：不碰术语，先 get 到装配流编程在干什么。

## 一个类比

盖房子有两种方式。

一种是让工人边想边砌墙——快，但你不知道他下一铲会怎么砌，返工了也说不清哪里错。

另一种是先让设计师画一张**施工图**，图纸经人审核，再交给施工队**照图施工**。施工队不即兴发挥，照着图，谁来盖、盖几次，结果都一样。

**装配流编程（AFP）选第二种。**

- AI 扮演设计师：理解需求，画出一张可审核的**蓝图**。
- 引擎扮演施工队：照蓝图**确定性地**拼装出软件。运行时没有 AI 参与。

## 一张图

```mermaid
flowchart LR
    需求[业务需求] --> AI[AI · 设计期<br/>画蓝图]
    AI --> 蓝图[蓝图<br/>可审核 · 可复现]
    蓝图 --> 引擎[引擎 · 运行期<br/>照图拼装 · 零 AI]
    引擎 --> 软件[软件<br/>结果确定]
```

## 它和"让 AI 直接写代码"差在哪

| | 让 AI 直接写代码 | AFP |
| :--- | :--- | :--- |
| AI 的产物 | 直接是代码 | 一张可审核的蓝图 |
| 谁来执行 | AI 边想边写 | 引擎照蓝图确定性拼装 |
| 能复现吗 | 难，每次可能不同 | 能，同蓝图同结果 |
| 出错能查吗 | 难，藏在生成的代码里 | 易，蓝图看得见 |

一句话：**别人让 AI 猜，AFP 让 AI 填图。**

## 一句话定义

> 用可复用的"装配块"，经"转接件"按"配置"驱动、对"数据"操作，装配成业务的"装配流"。AI 负责发现与配置，引擎负责确定性执行。
