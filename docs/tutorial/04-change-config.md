# 只改配置就改了行为

> 本课目标：亲眼看到——改一个配置值、不碰代码，行为就变了。

## 做一个实验

打开 `src/configs/register.jsonc`，把 `notifyChannel` 从 `"email"` 改成 `"sms"`：

```jsonc
"notifyChannel": "sms",
```

再跑一次：

```powershell
npm run assemble
```

观察输出里 `sent` 字段：

```json
"sent": [{ "channel": "sms", "to": "test@example.com", "message": "欢迎注册！" }]
```

通知走了短信。**你没改任何一个块的代码，没改拼装脚本。**

## 为什么这很重要

- 在传统开发里，改通知渠道 = 改代码 → 改测试 → 发版。
- 在 AFP 里，改通知渠道 = 改配置里一个值 → 重新拼装 → 行为变了。

这就是 AFP 甜区的核心卖点：**把"改行为"的成本从"改代码"降到"改配置"。**

## 但有前提

这之所以能行，是因为通知渠道是**枚举**（email / sms，有限二选一）。枚举路由可以留在配置里。

如果你想改的是"折扣规则的计算逻辑"——那是算法，不能放配置，就回到了"改代码"。这是 AFP 的边界，第 08 课会讲。

## 冒烟测试验证

不想手动试？跑测试，第 2 条冒烟就是自动验证这件事的：

```powershell
npm test
```

看到 `只改配置（email→sms），不改块代码，通知渠道就变了 ← 核心证据` 通过。
